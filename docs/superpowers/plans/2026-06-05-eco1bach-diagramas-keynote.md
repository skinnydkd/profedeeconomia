# Eco 1BACH — 8 diagrames keynote · Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Create 8 new SVG diagram components and mount them in Eco 1BACH units so its slide decks reach EDMN's diagram density (13 → 21).

**Architecture:** Each diagram is a self-contained `.astro` file holding one pure `<svg>`, following the `OfertaDemanda.astro` / `PesoMuerto.astro` pattern (design-system palette, Switzer/Fraunces/JetBrains Mono fonts, accessible `<title>`/`<desc>`, optional `<title>` tooltips on shapes). The name→component dispatch for decks (`SlideDiagramMount.astro`) is auto-generated by scanning the diagrams folder. Each diagram is mounted in its unit's MDX via the existing `<Diagram caption>` wrapper. No engine or archetype changes.

**Tech Stack:** Astro 5, hand-authored SVG, `scripts/gen-slide-diagram-mount.mjs`, `scripts/build-deck-pdf.mjs` (Puppeteer + overflow guard).

## Shared contract (every diagram component)

Read `src/components/diagrams/OfertaDemanda.astro` and `PesoMuerto.astro` first — they are the canonical references. Every new component MUST:

- Be `src/components/diagrams/<Name>.astro` with a leading `---` frontmatter holding only a JSDoc comment (no props).
- Contain exactly one `<svg viewBox="0 0 W H" xmlns="http://www.w3.org/2000/svg" role="img" aria-labelledby="<id>-title <id>-desc">` with `<title id>` and `<desc id>` as first children (Spanish, describing what the diagram shows).
- Define a `<style>` inside `<defs>` using the house palette ONLY:
  - bg `#FBF6EC`, ink `#2A1F18`, ink-soft `#5C4A3D`, ink-mute `#8A7868`, line `#E5D4BD`
  - terracota `#C44E2C` (+ deep `#9C3A1C`, soft `#FBE3D6`), mustard `#D4A24C` (+ deep `#A87A2A`, soft `#F5E5BC`)
  - teal `#1F6E6E` (+ soft `#DBEDED`) for a third series when needed
  - Convention already in use: **demand = terracota, supply/oferta = mustard**. Keep it.
  - Fonts: axis labels `'Switzer'` 700 uppercase letter-spaced; curve/point labels `'Fraunces'` italic; numeric ticks `'JetBrains Mono'`.
- Have a first full-canvas `<rect class="bg" .../>`.
- Be print-safe and fit a 16:9 slide at 1280×720 without overflow (the overflow guard enforces this). Prefer landscape `viewBox` around `800×500` and keep all content inside the box with comfortable margins.
- HTML comments `<!-- -->` are allowed (these are `.astro`, not MDX).

After creating a component file, the dispatch must be regenerated:

```bash
node scripts/gen-slide-diagram-mount.mjs
```

Expected: `✓ SlideDiagramMount.astro generated with N diagrams.` (N grows by 1 each time).

Mounting in MDX uses the existing wrapper (no import needed in MDX — diagrams resolve through the layout's component scope; verify the unit already uses `<Diagram>` and copy its usage):

```mdx
<Diagram caption="<peu explicatiu en castellà>">
  <NombreDiagrama />
</Diagram>
```
