# Insider (multi en xarxa) + real-time games framework — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Build the first networked-multiplayer game (Insider) plus the reusable real-time games framework atop PartyKit (Cloudflare Durable Objects).

**Architecture:** Two new pieces. **PartyKit server** at `party/insider/server.ts` (deployed independently to Cloudflare) holds the authoritative state per room (a Durable Object). **Astro client** has two entries: `/juegos/insider/host/` (teacher projector) and `/juegos/insider/` (student mobile), connected via WebSocket through PartyKit's TypeScript client. The server sends **private** messages per connection (role + word) and **public broadcasts** (phase + speaker + scoreboard). Shared TypeScript types in `src/lib/games-multi/insider/types.ts`. Pure game logic (scoring, role assignment, vote tally) lives in `party/insider/*.ts` as functions, testable with vitest.

**Tech Stack:** Astro 5, Preact (UI), TypeScript strict, vitest, **PartyKit** (new), **partysocket** client. Site stays on Vercel; PartyKit deploys to Cloudflare. New env var `PUBLIC_PARTYKIT_HOST`.

**Reference spec:** `docs/superpowers/specs/2026-05-26-insider-design.md`. **Branch:** `feat/insider`.

## Conventions

- Path alias `@/*` → `src/*`; `@components/*`, `@layouts/*`. NO `@lib`.
- Preact island pattern matches `src/components/games/stonks/*.tsx` (`/** @jsxImportSource preact */`, `class`, hooks).
- Content castellano; code English. Conventional Commits.
- **All randomness via injectable rng** in pure server functions (deterministic tests).
- Player IDs: UUID v4 generated client-side on first join, stored in `localStorage` as `pde:multi:playerId` for reconnect.
- Room codes: 4 chars from `ABCDEFGHJKLMNPQRSTUVWXYZ23456789` (no confusable 0/O/I/1).

```ts
import { WORDS } from './words';
export function assignRoles(playerIds: string[], impostorCount: number, rng: () => number) {
  const word = WORDS[Math.floor(rng() * WORDS.length)];
  const shuffled = [...playerIds].sort(() => rng() - 0.5);
  const impostors = new Set(shuffled.slice(0, impostorCount));
  return { impostors, word };
}
```

## Task 6: Typed client wrapper

**Files:** `src/lib/games-multi/insider/client.ts`

- [ ] **Step 1: Implement** a small wrapper around `partysocket` that:
  - Takes `{ host, roomCode, playerId, name, asHost }` and returns a connected client with typed `send(msg: ClientMsg)` and `on<T extends ServerMsg['type']>(type: T, handler: …)`.
  - Auto-reconnects on disconnect.
  - Exposes `state.public` and `state.private` reactive accessors (use a simple subscriber pattern).

```ts
import PartySocket from 'partysocket';
import type { ClientMsg, ServerMsg, PublicState, PrivateState } from './types';

export function createInsiderClient(opts: {
  host: string; roomCode: string; playerId: string; name: string; asHost?: boolean;
}) {
  const url = new URLSearchParams({ name: opts.name, asHost: opts.asHost ? '1' : '0' });
  const ws = new PartySocket({
    host: opts.host,
    party: 'insider',
    room: opts.roomCode,
    query: { playerId: opts.playerId, ...Object.fromEntries(url) },
  });
  const listeners = new Map<ServerMsg['type'], Set<(msg: any) => void>>();
  ws.addEventListener('message', (e) => {
    try {
      const msg: ServerMsg = JSON.parse(e.data);
      listeners.get(msg.type)?.forEach((l) => l(msg));
    } catch {}
  });
  ws.addEventListener('open', () => {
    ws.send(JSON.stringify({ type: 'join', name: opts.name, playerId: opts.playerId, asHost: opts.asHost ?? false } satisfies ClientMsg));
  });
  return {
    send(msg: ClientMsg) { ws.send(JSON.stringify(msg)); },
    on<T extends ServerMsg['type']>(type: T, h: (msg: Extract<ServerMsg, { type: T }>) => void) {
      let set = listeners.get(type); if (!set) { set = new Set(); listeners.set(type, set); }
      set.add(h as any); return () => set!.delete(h as any);
    },
    close() { ws.close(); },
  };
}
```
